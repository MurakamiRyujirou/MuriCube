# Docs/Domain_Cube.md

## 1. 概要
3D空間における「ブロックの集合体」の移動・回転・配色管理を定義する。
- **BlockGroup**: Block の集合の管理に徹し、座標と Block の対応を保持する（回転は持たない）。
- **Cube**: BlockGroup を保持するクラスであり、回転（Rotate）を担当する。
- **回転記法**: ルービックキューブの回転操作（R, U, F など）は**公式の記法**に準拠する。各面は「その面の外側から中心に向かって見て時計回り」を Clockwise とする（詳細は 4.1.1）。
特定の形状（テトリミノ等）に依存せず、任意の中心座標（Pivot）に基づいた空間回転をサポートする。

## 2. 基底型（Enums & Value Objects）

### 2.1 RotateAxis
- 実装では `Domain.Cube.Enums.RotateAxis`（X, Y, Z）。旧ドキュメント上の「CubeAxis」と同一の意味。

### 2.2 CubeTurn
- Clockwise (90°), CounterClockwise (-90°), HalfTurn (180°)

### 2.3 PivotPosition (Value Object)
- 回転の軸となる空間上の絶対位置を示す座標 (float X, float Y, float Z)。
- 座標系は 4.3 のとおり、**BlockPosition はブロックの中心**を表す。PivotPosition は同じ座標系における回転の中心点である。
- キューブ全体の中心で回転する場合の Pivot の例：
  - $1 \times 1 \times 1$：唯一のブロックの中心なので $(0, 0, 0)$。
  - $2 \times 2 \times 2$：ブロックが接する点（4 ブロックの境界）が中心なので $(0.5, 0.5, 0.5)$。
  - $3 \times 3 \times 3$：中央ブロックの座標が中心なので $(1, 1, 1)$。

## 3. 主要エンティティ

### 3.1 Block (Value Object) : IBlock 実装
- **データ構造**: 各 `BlockFace` に対応する `BlockColor` を保持。
- **責務**: `Rotate(RotateAxis axis)` により、その軸まわりの**時計回り 90°** に相当する面スワップだけを行い、新しい `Block` を返す。`CubeTurn`（180°・反時計回り）は `Cube` 側で `Rotate` を複数回合成して表現する。
- **インターフェース実装**: `GetColor(BlockFace face)` により指定面の配色を返す。

### 3.2 BlockGroup (Entity) : IBlockGroup 実装
複数の `Block` の集合の管理に徹する。回転（Rotate）は持たない。
- **データ構造**:
    - `Dictionary<BlockPosition, Block>` : 座標とブロック実体のマッピング。
- **インターフェースプロパティ**:
    - `IReadOnlyDictionary<BlockPosition, IBlock> Blocks` : 上記マッピングを IBlock として公開。
- **責務**: Block の集合とその配置構造の保持・公開のみ。回転ロジックは `Cube` が担当する。

### 3.3 Cube (Entity)
`BlockGroup` を保持し、グループ全体の回転を担う。
- **データ構造**:
    - `BlockGroup` : ブロック集合体への参照（または保持）。
    - `IBlockGroup BlockGroup` : 外部への読み取り専用公開。
- **回転・問い合わせ**:
    - `Rotate(RotateAxis axis, CubeTurn turn, PivotPosition pivot)` : 回転を実行し、新しい `Cube` を返す（不変操作）。
    - `GetAffectedBlocks(RotateAxis axis, CubeTurn turn, PivotPosition pivot)` : 境界比較に基づき、回転対象となる `BlockPosition` の集合を返す（View の `RotateAsync` 用）。
    - `GetPositionMap(RotateAxis axis, CubeTurn turn, PivotPosition pivot)` : **現在の** `Cube` の状態を前提に、1 手の回転を適用したあとの座標へ対応付ける「旧座標 → 新座標」マップだけを返す（回転対象外は含めない）。Presentation では **`Rotate` を呼ぶ前**のインスタンスで取得し、`Refresh` の `group` には **`Rotate` 後**の `IBlockGroup` を渡す。
    - `CanRotate(RotateAxis axis, CubeTurn turn, PivotPosition pivot)` : 回転後の座標が、静止ブロックまたは回転対象同士と**空間上重ならない**か検証する（中心間距離が全軸で 1 未満を重なりとみなす）。
- **回転ロジック（Rotate 内）**:
    - **公転**: 回転軸と Pivot から定まる「回転対象レイヤー」に属するブロックのみ、`pivot` を中心として `BlockPosition` を幾何学的に置換する。対象外のブロックは位置を変えない。
    - **自転**: 公転対象となったブロックに対して、`CubeTurn` に応じて `Block.Rotate` を複数回合成した結果で面スワップを反映する。対象外のブロックは色・向きを変えない。
    - **Z 軸の自転と公式記法の整合**: 公転の計算は `turn` のまま行う。`RotateAxis.Z` のときだけ、**自転**に渡す向きを `InvertTurn(turn)` で入れ替える（`Clockwise` ↔ `CounterClockwise`、`HalfTurn` は据え置き）。これにより、右手座標系・Unity の前向き（+Z）とドキュメント上の「正面から見た F の Clockwise」の整合を取る。Presentation の `CubeUIView` でも Z 軸の DOTween 回転角を同趣旨で符号反転している。
- **不変性**: `Rotate` は常に新しい `BlockGroup`（およびそれを保持する新しい `Cube`）を生成して返す。

## 4. 回転の定義（直感的アルゴリズム）

### 4.1 回転の考え方
回転は「**公転**（ブロックの位置移動）」と「**自転**（ブロック自身の向き変化）」の2つで構成される。
Cube の `Rotate(axis, turn, pivot)` はこの2つを必ずセットで実行する（`axis` の型は `RotateAxis`）。

1. **公転**: `pivot` を回転の中心として、対象ブロックの `BlockPosition` を新しい座標に移す。
2. **自転**: 公転対象となった各 `Block` に対して、`CubeTurn` に応じて `Block.Rotate(axis)` を複数回合成し、面の色の向きを入れ替える（Z 軸のみ自転用の `turn` を反転してから合成——3.3 参照）。

回転の3要素は常にセットで扱う。

| パラメータ | 意味 | 例 |
|---|---|---|
| `RotateAxis` | 回転の軸の向き | X（左右軸） |
| `CubeTurn` | 回転の方向と角度 | Clockwise（時計回り 90°） |
| `PivotPosition` | 回転の中心となる空間座標 | `(2.0, 2.0, 2.0)` |

### 4.1.1 回転記法の公式準拠（R, U, F など）

本プロジェクトで扱うルービックキューブの回転操作は、**公式の記法**に準拠する。

各面の回転は「その面の**外側から中心に向かって見て**時計回り」を **Clockwise** とする。

| 記法 | 対象面 | 見る向き | Clockwise の意味 |
|------|--------|----------|------------------|
| **R** | 右面 | 右側から中心に向かって見る | その向きで時計回りに 90° 回転 |
| **U** | 上面 | 上側から中心に向かって見る | その向きで時計回りに 90° 回転 |
| **F** | 正面 | 手前側から中心に向かって見る | その向きで時計回りに 90° 回転 |

- **R** は右面のブロックを、**右側から中心に向かって見て**時計回りに回転させる。実装では X 軸・対象レイヤー（例: 3×3 なら x=2）・`CubeTurn.Clockwise` で表す。
- **U** は上面のブロックを、**上側から中心に向かって見て**時計回りに回転させる。実装では Y 軸・対象レイヤー（例: 3×3 なら y=2）・`CubeTurn.Clockwise` で表す。
- **F** は正面のブロックを、**手前側から中心に向かって見て**時計回りに回転させる。実装では Z 軸・対象レイヤー（例: 3×3 なら z=0）・`CubeTurn.Clockwise` で表す。

L, D, B も同様に、それぞれ左面・下面・奥面を「その面から中心に向かって見て時計回り」を Clockwise として定義する。公転・自転の座標変換（4.2 以降）は、この約束に合わせてある。

### 4.2 公転：「Pivot からの相対位置」で移動先が決まる

公転は**回転対象レイヤーに属するブロック**にのみ適用する（4.6 実装指針で対象の決め方を定義する）。対象ブロックの移動先は次の手順で求める。

1. **相対座標の計算**: ブロックの位置から `pivot` を引き、「Pivot を原点とした相対座標」を得る。
2. **相対座標の回転**: 軸（`axis`）と方向（`turn`）に基づき、直交する2軸の値を入れ替える（後述）。
3. **絶対座標への復元**: 回転後の相対座標に `pivot` を足し戻し、新しい `BlockPosition` を得る。

**移動先は Pivot の位置によって変わる** ことに注意する。同じブロックでも、Pivot が異なれば移動先は異なる。

#### 軸ごとの相対座標の入れ替えルール（Clockwise 90° の場合）

実装の Rotate2D(Clockwise) は「新 a = b, 新 b = -a」で、軸ごとに次のように適用する。

X 軸回転（右側から見て時計回り）は、Y/Z 平面上で回転する。実装では `(ndz, ndy) = Rotate2D(dz, dy)` により、相対座標の変換は次のとおり。
- `新しい dy = -dz`
- `新しい dz = dy`

Y 軸回転（上から見て時計回り）は、X/Z 平面上で回転する。相対座標 `(dx, dz)` の変換は次のとおり。
- `新しい dx = dz`
- `新しい dz = -dx`

Z 軸回転（手前から見て時計回り）は、X/Y 平面上で回転する。相対座標 `(dx, dy)` の変換は次のとおり。
- `新しい dx = dy`
- `新しい dy = -dx`

CounterClockwise は Clockwise の逆変換、HalfTurn は Clockwise を2回適用した変換とする。

### 4.3 座標系とブロック位置の前提

BlockPosition はブロックの**中央**のワールド座標を指す（Unity の Transform.position と同じ仕様）。
サイズ 1 のブロックを `(x, y, z)` に置いた場合、そのブロックは空間上で `x-0.5 ～ x+0.5`、`y-0.5 ～ y+0.5`、`z-0.5 ～ z+0.5` の範囲を占める。

したがって「ブロックとブロックの境界（格子点）」は整数座標ではなく、**0.5 刻みの座標**に存在する。
- 例：`(3, 0.5, 0.5)` は X=3 の列において、ブロック同士が接する境界（格子点）を貫く軸の位置である。

**落下中ピースにおける座標の扱い**
BlockPosition はピース内部の相対座標であり、空中落下中はフィールドの絶対座標とは独立している。回転後に Y 座標がマイナスになることは許容される。ピースが空中にある間はどの座標にブロックが存在しても問題なく、ピースがフィールドに固定された際にフィールド側で範囲チェックを行う。

### 4.4 具体例：4×4×4 キューブの R 回転（実測値）

4×4×4 のキューブで、座標系は TechSpecs に従う（x: 右+, y: 上+, z: 奥+）。
**R 回転**は「X 軸正方向から見たときの Clockwise 90° 回転」とする。

以下、同じブロック `(3, 3, 3)` を3つの異なる Pivot で回した場合の移動先を示す。
数値は Unity 上で実測済み。

---

**例 1：Pivot = `(3.0, 0.5, 0.5)`（X=3 列の左下手前の格子点）で回す場合**

ブロックの角（格子点）を回転軸とするケース。4×4×4 の標準的な R 回転に相当する。

ブロック `(3, 3, 3)` の移動先：
- 相対座標: `(3-3.0, 3-0.5, 3-0.5)` = `(0.0, 2.5, 2.5)`
- X 軸 Clockwise 回転: `新dy = -dz = -2.5`、`新dz = dy = 2.5` → 相対 `(0.0, -2.5, 2.5)`
- 絶対座標に復元: `(3.0, -2.5+0.5, 2.5+0.5)` = `(3, -2, 3)`

`(3, 3, 3)` → `(3, -2, 3)` ✅（Unity 実測値と一致）

---

**例 2：Pivot = `(3.0, 1.5, 1.5)`（X=3 列の中央）で回す場合**

X=3 の1列のみが対象で、Y/Z 方向の中心を列の中央に置くケース。

ブロック `(3, 3, 3)` の移動先：
- 相対座標: `(0.0, 1.5, 1.5)`
- X 軸 Clockwise 回転: `新dy = -1.5`、`新dz = 1.5` → 相対 `(0.0, -1.5, 1.5)`
- 絶対座標に復元: `(3.0, -1.5+1.5, 1.5+1.5)` = `(3, 0, 3)`

`(3, 3, 3)` → `(3, 0, 3)`

---

**例 3：Pivot = `(2.0, 2.0, 2.0)`（キューブ全体の中央）で回す場合**

4×4×4 全体の空間中心を軸とするケース。X=2 と X=3 の2列が回転対象となる。

ブロック `(3, 3, 3)` の移動先：
- 相対座標: `(1.0, 1.0, 1.0)`
- X 軸 Clockwise 回転: `新dy = -1.0`、`新dz = 1.0` → 相対 `(1.0, -1.0, 1.0)`
- 絶対座標に復元: `(3.0, -1.0+2.0, 1.0+2.0)` = `(3, 1, 3)`

`(3, 3, 3)` → `(3, 1, 3)`

---

**3例の比較まとめ**

| Pivot | 回転対象のレイヤー | `(3,3,3)` の移動先 |
|---|---|---|
| `(3.0, 0.5, 0.5)` | X=3 の1列（格子点軸） | `(3, -2, 3)` |
| `(3.0, 1.5, 1.5)` | X=3 の1列（列中央軸） | `(3, 0, 3)` |
| `(2.0, 2.0, 2.0)` | X=2 と X=3 の2列 | `(3, 1, 3)` |

このように、**Pivot の Y/Z 値**が「どこを中心に回すか」を決定し、移動先が変わる。
同じブロックでも Pivot が異なれば移動先は完全に異なる。

### 4.5 自転：ブロック自身の向きの変化

公転によってブロックの位置が移動すると同時に、そのブロック自身も同じ軸・同じ方向に向きを変える（自転する）。
自転は `Cube` が `CubeTurn` に応じて `Block.Rotate(RotateAxis)` を合成して行い、6面の色が面スワップのルールで入れ替わる。

**X 軸 Clockwise 回転（R 回転）における面スワップの例**：

| 移動前の面 | 移動後の面 |
|---|---|
| 手前面（Front） | 上面（Top） |
| 上面（Top） | 奥面（Back） |
| 奥面（Back） | 下面（Bottom） |
| 下面（Bottom） | 手前面（Front） |
| 左面・右面 | 変化なし |

このルール（各軸・各方向ごとの面スワップ対応表）の詳細は TechSpecs 3.2 に定義する。

> **具体例**: R 回転（Pivot=`(3.0, 0.5, 0.5)`）で `(3, 3, 3)` から `(3, -2, 3)` へ移動したブロックは、移動後に「正面にあった色が上面として現れる」状態になる。

### 4.6 実装指針

- **回転対象の決定（境界比較）**
  - 「四捨五入して一致する層」ではなく、**CubeTurn に応じた境界比較**で回転対象を決める。Clockwise は正方向側、CounterClockwise は負方向側を対象とする。
  - **X 軸**: CW → `Block.X > pivot.X`（Pivot より右）、CCW → `Block.X < pivot.X`（Pivot より左）。
  - **Y 軸**: CW → `Block.Y > pivot.Y`（Pivot より上）、CCW → `Block.Y < pivot.Y`（Pivot より下）。
  - **Z 軸**: CW → `Block.Z < pivot.Z`（Pivot より手前）、CCW → `Block.Z > pivot.Z`（Pivot より奥）。Z 軸の正負はプロジェクトの座標系に合わせる。
  - HalfTurn は CW と同じ側を対象とする。
  - 例：Pivot を (1, 0.5, 0.5) に統一したとき、R（X軸 CW）は `pos.X > 1` のブロック、L（X軸 CCW）は `pos.X < 1` のブロックが対象になる。

- **GetAffectedBlocks / GetPositionMap**
  - `Cube` に実装済み。`GetAffectedBlocks` は View の `RotateAsync` に渡す座標リスト用。`GetPositionMap` は `Refresh` 時に「移動したブロックの View」を新しい `BlockPosition` にひも付け直すために呼び出し側が渡す。

- **公転（対象ブロックの座標の移動）**
  1. 上記で得た各対象 Block の座標から Pivot を引き、相対座標を求める。
  2. 4.2 公転のルールに従い、相対座標の直交2軸を入れ替える。
  3. Pivot を加算して新しい絶対座標を得る。BlockPosition は float のため丸めずにそのまま返す（0.5 刻み Pivot で非整数座標を許容）。

- **自転（対象ブロックの向きの更新）**
  - 公転対象となった各 Block に対してのみ、実装どおり `CubeTurn` に応じた `Block.Rotate(axis)` の合成（Z 軸は `InvertTurn`）で新しい Block を得る。
  - TechSpecs 3.2 の規則（X/Y/Z ごとの面スワップ）をそのまま適用する。

- **新しい BlockGroup の構築**
  - 回転対象ブロックは、上記で得た**新しい座標**と**自転後の Block** をマッピングに含める。
  - 回転対象外のブロックは、**元の座標**と**元の Block**（自転させない）をそのままマッピングに含める。
  - 以上で新しい `BlockGroup` を構築し、それを保持する新しい `Cube` を返す。

このように、Cube の Rotate は「対象レイヤーのみ」公転・自転させ、それ以外は変更しない設計とする。

## 5. 設計指針
- **エンジンの非依存性**: `UnityEngine` を含む外部ライブラリに依存せず、純粋な C# の論理演算として実装する。
- **形状の自由度**: ブロックがどのように連結されていても、指定された `pivot` を基準に整合性を保って回転可能とする。回転後に形状が変化するのは仕様通りの正しい挙動であり、落下ピースが固定されるまでの間は任意の形状で落下を続ける。
- **副作用の排除**: 状態の上書きを禁止し、常に新しいインスタンスを構築することで、データの整合性を担保する。
