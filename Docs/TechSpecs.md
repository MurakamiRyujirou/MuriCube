# Docs/TechSpecs.md

## 1. 座標系とデータ構造の定義

### 1.1 グリッド座標系
- 単位: 1.0 単位の整数グリッド（Unityの座標系に準拠）。
- 原点 (World Origin): フィールドの「左下手前」を (0, 0, 0) とする。
- 軸の定義:
    - x軸: 右方向がプラス (0 〜 9)
    - y軸: 上方向がプラス (0 〜 19)
    - z軸: 奥方向がプラス (0: Frontレイヤー, 1: Backレイヤー)

### 1.2 ブロックデータの属性定義
各ブロックは、以下の6方向に対応する配色情報を保持する。

- 方向定義 (FaceDirection):
    - Up, Down, Left, Right, Front (z=0方向), Back (z=1方向)
- 配色定義 (ColorID):
    - Red, Blue, White, Yellow, Green, Orange
    - (Empty: ブロックが存在しない状態)

## 2. 回転ロジック (Rotation Logic)

### 2.1 90度単位の座標置換（2x2x2内での位置変更）
回転軸を中心とした 2x2x2 の範囲内において、各ブロックのインデックス (x, y, z) は以下のルールで入れ替わる。

- **X軸回転 (Right/Left面を回す操作)**:
    - 横(x)の位置は固定。
    - 縦(y)と奥行き(z)が入れ替わる。 (y, z) -> (1-z, y) ※時計回りの場合
- **Y軸回転 (Up/Down面を回す操作)**:
    - 縦(y)の位置は固定。
    - 横(x)と奥行き(z)が入れ替わる。 (x, z) -> (z, 1-x)
- **Z軸回転 (Front/Back面を回す操作)**:
    - 奥行き(z)の位置は固定。
    - 横(x)と縦(y)が入れ替わる。 (x, y) -> (1-y, x)

### 2.2 ブロック内配色データのスワップ
位置の移動と同時に、ブロック自体が持つ6面の配色情報を以下の通り更新する。

- **X軸回転**: 
    - Front の色が Down へ
    - Down の色が Back へ
    - Back の色が Up へ
    - Up の色が Front へ
    - (Left / Right は不変)

- **Y軸回転**:
    - Front の色が Left へ
    - Left の色が Back へ
    - Back の色が Right へ
    - Right の色が Front へ
    - (Up / Down は不変)

- **Z軸回転**:
    - Up の色が Right へ
    - Right の色が Down へ
    - Down の色が Left へ
    - Left の色が Up へ
    - (Front / Back は不変)

## 3. 回転軸の動的制御 (Dynamic Pivot)

- **軸の選択**: 
    - プレイヤーの入力により、x軸上の隣り合うグリッド境界を回転軸（2x2x2のケージ中心）として切り替える。
- **回転対象**:
    - 選択された軸を中心とした 2x2x2 空間内に存在する全てのブロック。

## 4. 消去判定と更新処理

1. **行走査**: y=0 から順に y=19 まで調査。
2. **充填確認**: Frontレイヤー (z=0) において、対象行の x=0〜9 全てにブロックが存在するか。
3. **色一致確認**: 対象行の全ブロックの「Front」面の色が、10マス全て同一か。
4. **消去実行**: 
    - 条件を満たした行のブロックを削除。
    - その行より上の全ブロックを y-1 下降させる。
    - Backレイヤー (z=1) にあるブロックは、接地固定時に一括で破棄する。

## 5. 実装上の注意
- **Unity非依存の維持**: ドメインロジック（回転・消去判定等）は UnityEngine 名前空間に依存させないこと。
- **計算精度**: 独自の 3D整数座標クラス（例: FieldPosition 等）を使用し、浮動小数点による演算を完全に排除する。
- **不変性の維持**: 回転操作時は新しい配列およびブロック状態を生成して返す不変（Immutable）操作を基本とする。
