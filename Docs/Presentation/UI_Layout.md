# Docs/Presentation/UI_Layout.md

## 1. 概要

MuriCube の画面は iPhone 縦持ちを基本とし、**2つの表示エリア**で構成される。
2つのエリアは同一の `ActiveMino` を参照し、常に同期する。

## 2. カメラ構成

**1台のカメラ**で3D空間全体を描画する。フィールドエリアとキューブエリアは同一の3D空間内に配置し、
ワールド座標の Y 軸方向で上下に分離する。

| エリア | ワールド座標の範囲 | 説明 |
|--------|------------------|------|
| フィールドエリア | Y: 0〜19、X: 0〜9、Z: 0 | テトリスのフィールド（10×20グリッド） |
| キューブエリア | Y: 22〜30付近 | ActiveMinoを立体表示する浮遊エリア |

カメラは斜め上方からやや俯瞰した角度に設定し、フィールドとキューブエリアの両方が映るよう調整する。

## 3. 画面構成

```
┌─────────────────────────┐
│                         │
│     キューブエリア         │  Y=22〜30付近
│   （ActiveMino 3D表示）   │
│   [楕円UI・角丸四角UI]     │
│                         │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │  Y=20（境界）
│                         │
│     フィールドエリア        │  Y=0〜19
│   （積み上がりブロック）     │
│   （落下中ミノ z=0面）      │
│                         │
│  [←] [→]   [↓] [⬇⬇]  │
└─────────────────────────┘
```

## 4. キューブエリア

### 4.1 表示仕様

- **表示対象**: 落下中の `ActiveMino` のみ。フィールドの積み上がりブロックは表示しない。
- **配置**: フィールドエリアの上方（Y=22〜30付近）に `CubeUIView` を配置する。
- **落下**: このエリアではミノは落下しない。プレイヤーがルービックキューブ操作に集中できる固定表示。
- **同期**: フィールドエリアの `ActiveMino` と常に同期し、回転・移動が即座に反映される。

### 4.2 回転操作UI（ルービックキューブ操作パーツ）

ミノの3D表示に重ねて、以下のUIパーツを配置する。

#### 楕円パーツ（R/L/U/D 回転）

| 配置 | 形状 | スワイプ操作 |
|------|------|------|
| 上辺 | 横長楕円（1×2相当） | 左スワイプ → U回転 / 右スワイプ → U'回転 |
| 下辺 | 横長楕円（1×2相当） | 左スワイプ → D回転 / 右スワイプ → D'回転 |
| 左辺 | 縦長楕円（2×1相当） | 上スワイプ → L回転 / 下スワイプ → L'回転 |
| 右辺 | 縦長楕円（2×1相当） | 上スワイプ → R回転 / 下スワイプ → R'回転 |

#### 角丸四角パーツ（F/B 回転）

| 配置 | サイズ | タップ操作 |
|------|------|------|
| 左下 | 1×1角丸四角 | シングルタップ → F回転 / ダブルタップ → F'回転 |
| 右上 | 1×1角丸四角 | シングルタップ → B回転 / ダブルタップ → B'回転 |

## 5. フィールドエリア

### 5.1 表示仕様

- **表示対象**: 積み上がったブロック（`Field`）と落下中の `ActiveMino`（z=0面のみ）。
- **座標**: ドメインの `CubePosition(x, y, 0)` をそのまま `Vector3(x, y, 0)` にマッピングする。
- **ActiveMino の表示**: z=0面の色を表示する。キューブエリアで回転操作が行われると色がリアルタイムで更新される。

### 5.2 テトリス操作ボタン

フィールドエリア下部に固定ボタンを配置する。

| ボタン | 操作 |
|------|------|
| ← | 左移動（X-1） |
| → | 右移動（X+1） |
| ↓ | ソフトドロップ（Y-1、押し続けで加速） |
| ⬇⬇ | ハードドロップ（即着地） |

## 6. デバイス別入力対応

### 6.1 iPhone タッチ

| 操作 | 入力 |
|------|------|
| R/L/U/D 回転 | キューブエリアの楕円パーツをスワイプ |
| F 回転 | 左下角丸四角をシングルタップ |
| F' 回転 | 左下角丸四角をダブルタップ |
| B 回転 | 右上角丸四角をシングルタップ |
| B' 回転 | 右上角丸四角をダブルタップ |
| 左移動 | フィールドエリア ← ボタン |
| 右移動 | フィールドエリア → ボタン |
| ソフトドロップ | フィールドエリア ↓ ボタン |
| ハードドロップ | フィールドエリア ⬇⬇ ボタン |

### 6.2 キーボード

| 操作 | キー |
|------|------|
| R/U/F/L/D/B 回転（時計回り） | 対応キー |
| 逆回転（反時計回り） | Shift + 対応キー |
| 左移動 | ← |
| 右移動 | → |
| ソフトドロップ | ↓ |
| ハードドロップ | Space |

### 6.3 ゲームパッド

#### 回転操作

| 操作 | ボタン |
|------|------|
| R 回転 | 右ボタン右（Bボタン相当） |
| U 回転 | 右ボタン上（Yボタン相当） |
| F 回転 | 右ボタン下（Aボタン相当） |
| B 回転 | DPad上 |
| L 回転 | DPad左 |
| D 回転 | DPad下 |
| 逆回転 | 左トリガー または 右トリガー + 対応ボタン |

#### テトリス操作

| 操作 | ボタン |
|------|------|
| 左移動 | 左スティック左 |
| 右移動 | 左スティック右 |
| ソフトドロップ | 左スティック下 |
| ハードドロップ | 右ボタン左（Xボタン相当） |

## 7. 2エリアの同期仕様

- `ActiveMino` は Application 層の `GameState` で単一管理される。
- `CubeUIView`（キューブエリア）と `FieldUIView`（フィールドエリア）は、どちらも `GameStateMachine.GameStateObservable` を R3 で購読する。
- 回転操作により `GameState.ActiveMino` が更新されると、両エリアが同時に再描画される。
