# Docs/Presentation/UI_Layout.md

## 1. 概要

MuriCube の画面は iPhone 縦持ちを基本とし、**2つの表示エリア**で構成される。
2つのエリアは同一の `ActiveMino` を参照し、常に同期する。

## 2. 画面構成

```
┌─────────────────────────┐
│  [設定]               ⚙️ │
├─────────────────────────┤
│                         │
│     キューブエリア         │  上部 約30%〜40%
│   （3D斜め視点表示）       │
│                         │
│  [楕円UI・角丸四角UI]      │  ← ミノに貼り付く回転操作パーツ
├─────────────────────────┤
│                         │
│                         │
│     フィールドエリア        │  下部 約60%〜70%
│   （正面カメラ・平面表示）   │
│                         │
│  [←] [→]   [↓] [⬇⬇]  │  ← テトリス操作ボタン
└─────────────────────────┘
```

## 3. キューブエリア

### 3.1 表示仕様

- **カメラ**: 斜め上方からの等角投影（または透視投影）。ActiveMino を3D表示する。
- **表示対象**: 落下中の `ActiveMino` のみ。フィールドの積み上がりブロックは表示しない。
- **落下**: このエリアではミノは落下しない。プレイヤーが配色操作に集中できる固定表示。
- **同期**: フィールドエリアの `ActiveMino` と常に同期し、回転・移動が即座に反映される。

### 3.2 回転操作UI（ルービックキューブ操作パーツ）

ミノの3D表示に重ねて、以下のUIパーツを配置する。パーツはミノの各面に貼り付くように配置され、視点に追従する。

#### 楕円パーツ（R/L/U/D 回転）

ミノのF面（z=0側）の外周に沿って4つの楕円を配置する。形状はミノのバウンディングボックスを基準に決定する。


| 配置  | 形状          | スワイプ操作                     |
| --- | ----------- | -------------------------- |
| 上辺  | 横長楕円（1×2相当） | 左スワイプ → U回転 / 右スワイプ → U'回転 |
| 下辺  | 横長楕円（1×2相当） | 左スワイプ → D回転 / 右スワイプ → D'回転 |
| 左辺  | 縦長楕円（2×1相当） | 上スワイプ → L回転 / 下スワイプ → L'回転 |
| 右辺  | 縦長楕円（2×1相当） | 上スワイプ → R回転 / 下スワイプ → R'回転 |


#### 角丸四角パーツ（F/B 回転）


| 配置  | サイズ     | タップ操作                         |
| --- | ------- | ----------------------------- |
| 左下  | 1×1角丸四角 | シングルタップ → F回転 / ダブルタップ → F'回転 |
| 右上  | 1×1角丸四角 | シングルタップ → B回転 / ダブルタップ → B'回転 |


#### パーツのスタイル

- 色: 白い細線（既存ルービックキューブアプリのデザインを踏襲）
- 背景: 透明
- ミノの形状が変わっても、バウンディングボックスを基準に自動的に位置を再計算する

## 4. フィールドエリア

### 4.1 表示仕様

- **カメラ**: 正面からの平面表示（orthographic）。
- **表示対象**: 積み上がったブロック（`Field`）と落下中の `ActiveMino`（z=0面のみ）。
- **ActiveMino の表示**: z=0面の色を平面で表示する。キューブエリアで回転操作が行われると、正面の色がリアルタイムで更新される。

### 4.2 テトリス操作ボタン

フィールドエリア下部に固定ボタンを配置する。


| ボタン | 操作                   |
| --- | -------------------- |
| ←   | 左移動（X-1）             |
| →   | 右移動（X+1）             |
| ↓   | ソフトドロップ（Y-1、押し続けで加速） |
| ⬇⬇  | ハードドロップ（即着地）         |

## 6. 2エリアの同期仕様

- `ActiveMino` は Application 層の `GameState` で単一管理される。
- キューブエリアの View とフィールドエリアの View は、どちらも `GameStateMachine.GameStateObservable` を R3 で購読する。
- 回転操作により `GameState.ActiveMino` が更新されると、両エリアが同時に再描画される。
- キューブエリアは `IBlockGroup` の全面の色を3D表示し、フィールドエリアは `IBlock.GetColor(BlockFace.Front)` のみを平面表示する。

- スコア・レベル表示
- DPad右の活用
- 画面の左右分割レイアウト（キューブエリアを左、フィールドエリアを右に配置する案）
